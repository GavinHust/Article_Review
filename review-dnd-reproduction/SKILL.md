# 审查 DND 论文复现

审查真实行为，不根据文件名或表面 API 相似性下结论。每个结论都必须形成“论文要求 → 代码/配置/数据/测试证据”的追踪链。

## 必要输入

获取待审查仓库或代码文件夹路径。默认以随附的 DND 规格作为论文事实来源；若用户提供其他论文版本，先提取并核对版本差异。

开始前读取：

- `references/dnd-paper-spec.md`：论文明确要求及歧义边界。
- `references/review-rubric.md`：严重级别、置信度和审查步骤。
- `references/authorship-and-understanding.md`：AI 生成风险、理解深度与独立思考的谨慎评价方法。
- `assets/report-template.md`：最终报告模板。

## 工作流程

1. 盘点仓库。
   - 运行 `python scripts/audit_repo.py <仓库> --output <inventory.json>`。
   - 检查生成的索引、仓库说明、README、依赖文件、配置、数据脚本、模型代码、训练/评估入口、测试、日志和结果产物。
   - 索引只用于导航；必须打开实际实现再判断。
2. 建立追踪矩阵。
   - 将论文规格中的每项适用要求映射到准确的 `文件:行号` 证据。
   - 状态使用：`符合`、`部分符合`、`不符合`、`缺失`、`无法运行`、`论文未规定`。
   - 区分实际实现、README 声明和已有结果产物。
3. 优先重建实际模型结构。
   - 沿训练入口追踪最终实例化的模型和配置，不能只读类定义或默认参数。
   - 输出逐层结构表：层名/类型、输入维度、输出维度、head 数、拼接或平均、激活、dropout、残差/归一化、参数量、论文对应项、差异与影响。
   - 用真实样本或最小图记录关键张量 shape；核对多头拼接后的实际维度、残差投影维度和 MLP 输入维度。
   - 明确列出 loss 名称、公式或框架调用、输入是 logits 还是 sigmoid 后概率、标签类型/范围、reduction、类别权重，以及它与论文软标签的兼容性。
   - 论文没有给出 loss 名称时，评价其合理性与披露程度，但不得仅因不同于猜测实现而判错。
4. 深入检查其余高风险路径。
   - 数据：ER/SBM/SPL 参数、图规模和数量、自环、边方向、训练/测试隔离、特征重算。
   - 标签：穷举枚举、最小节点数标准、LSCC 0.1 阈值、全部并列最优集合、软成员频率标签。
   - 模型：入邻居聚合、四层 GAT、每层通道和 head、多头合并、线性残差、ELU、MLP 各层维度、Sigmoid、参数初始化和前向 shape。
   - 训练：不同网络族的超参数、50 个 epoch、loss 与软标签兼容性、batch、数据泄漏、checkpoint 选择、可复现控制。
   - 测试：DND 静态排序与自适应基线、删除节点及关联边、逐步 LSCC、AUC 定义和归一化、十实例聚合、随机重复、规模网格、计时边界、消融和真实网络。
5. 执行适度验证。
   - 先运行已有单元测试和文档中的最小命令。
   - 用户只要求 review 时不要修改仓库；必要探针放在仓库外的临时目录。
   - 优先用小型确定性有向图验证边方向、SCC、穷举标签、并列最优处理、`相对 LSCC <= 0.1` 停止条件和 AUC 索引。
   - 对 1K–100K 的昂贵实验，先验证配置并运行最小代表案例，明确标记未执行的大规模声明。
6. 质疑结果链。
   - 若有原始删除顺序或日志，至少独立重算一个指标。
   - 检查 DND 耗时是否包含预测和拆解、排除训练，与论文一致。
   - 检查比较中是否混用了归一化/原始 AUC、平均值/十次最佳值、静态/自适应排序。
   - 将复现结果与论文的定性结论逐项对照；有原始数据时报告绝对值、相对差异、方差和统计口径。
   - 检查曲线、表格和日志能否从当前代码与配置重新生成，防止只提交无法溯源的结果图片。
7. 评价复现质量、理解和原创性风险。
   - 按 `references/authorship-and-understanding.md` 检查解释、实验设计、边界条件、消融、失败分析和代码/结果溯源。
   - 分别列出有证据的优点与缺点，不以“没有发现错误”代替优点评价。
   - AI 生成只给风险等级和可核验迹象；至少需要多类相互印证的证据，不能根据注释多、命名整齐或代码风格单独判断。
8. 先报告发现。
   - 按严重程度列出具体问题，再列覆盖范围和受阻检查。
   - 每项包含论文主张、代码证据、实际影响和最小验证/修复方向。
   - 论文只有图或归一化柱状值时，不得宣称精确数值复现。

## 判断规则

- 仅当代码或实测行为违背论文明确主张时，判为 `不符合`。
- 只有在搜索入口、配置、辅助函数和测试后仍未发现必要行为，才判为 `缺失`。
- 论文未交代的选择标为 `论文未规定`；可作为复现风险，但不能当作论文违例。
- 图语义和输出等价的替代实现应判为 `符合`。
- 只有 README 声明、没有代码/数据/命令/产物支持时，标为未验证。
- 只有直接代码/配置证据或成功的定向执行，才能给出高置信度。
- 区分忠实度与方法质量：忠实复现也可能继承论文自身的方法学缺陷。
- 区分“AI 辅助痕迹”和“低质量复现”：AI 辅助本身不是缺陷，未验证、伪造、无法解释或语义错误才是实质问题。
- 评价理解与思考时引用 README、注释、实验设计、commit/产物或实际实现证据，不揣测个人能力或动机。

## 最低输出要求

交付内容必须包括：

1. 总体结论；
2. 按严重级别排序、带可点击 `文件:行号` 的问题；
3. 论文模型与复现模型的逐层结构/维度/激活/loss 对照表；
4. 数据、标签、特征、baseline、训练、测试、指标和结果的符合矩阵；
5. 复现实验结果与论文结论的对照；
6. 有证据的复现优点与缺点；
7. 复现者理解与独立思考评价；
8. AI 生成风险等级、迹象、反证与判断局限；
9. 已执行命令、论文歧义、未执行检查和优先后续行动。

若没有发现可操作的不一致，要明确说明，并列出剩余验证缺口。

# AI 生成风险、论文理解与独立思考评价

## 原则

- 只能评价仓库和复现材料表现出的迹象，不能判定作者身份、动机或是否“作弊”。
- AI 辅助编程本身不是缺点。核心是实现是否正确、可解释、可验证、可追溯。
- 单一风格迹象没有证明力。至少两类独立证据相互印证，才给出中等或更高风险。
- 同时寻找反证，避免确认偏误。

## AI 生成风险等级

- `低`：代码和实验链条连贯，可解释关键设计，结果可重建；即使可能使用 AI，也没有明显质量风险。
- `中`：存在多处模板化/不一致迹象，但核心实现仍可运行，来源无法确定。
- `高`：多类证据共同显示内容未经核验或拼接生成，并实质影响可信度，例如虚构 API、不可达代码、伪造结果链。
- `无法判断`：证据不足。默认使用此结论，而不是猜测。

## 可核验迹象

以下只能作为迹象，不能单独定性：

- README/注释声称的层数、维度、命令或结果与实际代码明显不一致。
- 大量模板占位符、虚构依赖/API、未定义变量、从未被调用的“完整功能”。
- 同一概念在不同文件中术语、shape、框架或编码风格突然变化，疑似拼接且没有适配。
- 注释逐句复述代码但没有解释 DND 的入邻居、软标签、LSCC 等关键语义。
- 异常宽泛的异常捕获、硬编码假结果、静默 fallback、随机数冒充实验输出。
- 图表、日志、checkpoint 缺少生成命令和配置，时间戳/shape/数据规模互相矛盾。
- 测试只断言“能运行”，不验证论文核心语义；或测试内容与实现完全脱节。
- 提交历史若可用：一次性导入大量不连贯代码、随后只修语法错误。提交历史缺失不能作为负面证据。

反证包括：

- 有针对论文歧义的明确取舍说明和可复现实验。
- 有失败实验、调试记录、参数演化和原因分析。
- 有小图语义测试、shape trace、原始日志和结果生成链。
- 代码虽简洁或风格统一，但实际行为、测试和文档相互一致。

## 论文理解评价

按 `充分 / 基本 / 表面 / 无法判断` 评价，并逐项给证据：

1. 是否理解目标是有向图 LSCC，而非无向/WCC 或普通节点分类。
2. 是否理解标签来自全部最小基数拆解集合的成员频率，而非单一贪心答案。
3. 是否理解消息方向是入邻居聚合，并能解释框架 `edge_index` 约定。
4. 是否理解 DND 静态预测排序与自适应 baseline 的不同。
5. 是否理解 AUC、0.1 停止阈值、原始/归一化指标和随机重复口径。
6. 是否识别论文没有说明的 loss、生成器参数、AUC 离散约定等限制。

不能仅凭注释或 README 自述判为“理解充分”；必须与实现或实验相互印证。

## 独立思考评价

寻找复现者是否：

- 对论文歧义提出合理实现选择，并说明影响。
- 设计额外对照、消融、边界测试或失败案例。
- 分析复现结果与论文不同的原因，而不是只调参追图。
- 指出论文协议本身的局限，例如真实图 baseline 取十次最佳值的公平性。
- 讨论方法适用范围、计算复杂度、标签穷举扩展性或对图类型的敏感性。
- 提出可验证改进，而不是泛泛写“未来可优化”。

## 输出要求

分别输出：

1. AI 生成风险等级；
2. 支持迹象；
3. 反证；
4. 判断局限；
5. 论文理解等级及证据；
6. 独立思考亮点和欠缺。

不要使用“肯定是 AI 写的”“作者不懂论文”等确定性或人身化表述。

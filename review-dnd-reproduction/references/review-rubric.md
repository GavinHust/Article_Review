## 状态与严重级别

每项论文要求使用一个状态：

- `符合`：证据实现了论文主张或语义等价形式。
- `部分符合`：只覆盖部分必要情况或细节。
- `不符合`：直接违背明确论文主张。
- `缺失`：完整搜索后仍未找到必要路径。
- `无法运行`：存在预期实现，但当前环境不能验证。
- `论文未规定`：论文没有给出唯一预期实现。

严重级别：

- `P0`：核心任务失效或结果存在伪造/泄漏，例如用测试结果生成标签。
- `P1`：实质改变标签、有向聚合、LSCC 目标、模型或核心评估。
- `P2`：削弱可复现性，或改变部分实验、基线或指标。
- `P3`：影响有限的报告、健壮性或可维护性问题。

## 置信度

- `高`：直接行号证据加定向执行，或无歧义的配置/实现冲突。
- `中`：有直接静态证据但无法运行，或行为依赖已从本地文档/源码确认的库约定。
- `低`：仅根据命名、不完整产物或论文歧义推断。不得把低置信度判断写成事实。

## 审查顺序

1. 阅读仓库级说明，确认其声称的复现范围。
2. 定位数据生成、标签、模型、训练、评估、基线、指标和测试路径。
3. 沿实际入口追踪 import；未被调用的示例/废弃代码不能作为实现证据。
4. 核对配置默认值和 CLI 覆盖，记录最终生效值。
5. 查看持久化数据元信息和结果日志，确认实际使用的生成器和配置。
6. 从训练入口重建最终模型，运行一次 shape trace，记录实际层、维度和参数量。
7. 追踪 loss 从标签加载到反向传播的完整数据流，确认 logits/概率接口和软标签语义。
8. 运行已有测试，再用小图探针验证普通测试遗漏的语义。
9. 条件允许时独立重算一个样例结果。
10. 用准确证据撰写问题，不把结论扩大到未测试范围。

## 定向测试

### 有向 LSCC

构造弱连通与强连通结果不同的图，验证评估器使用强连通分量，并以原始 `N` 为分母。

### 入邻居聚合

使用非对称三节点图和不同 one-hot 特征。验证改变前驱会改变目标节点表示，而只改变其出边后继不会在同一聚合步影响目标节点。注意 GAT 库可能自动添加自环。

### 并列最优集合

构造或搜索至少有两个最小拆解集合的小型有向图，独立枚举组合，逐节点比较预期频率软标签，不能只比较二值支持集合。

### 阈值边界

测试 `LSCC/N` 恰好高于、等于和低于 0.1 的情况。对 `0.1N` 非整数的 `N` 检查取整规则。

### 删除曲线与 AUC

对已知删除顺序独立计算每次删除后的 SCC 大小。检查是否包含初始点和跨过阈值后的点；论文对此有歧义，应披露实现约定。

### 静态与自适应策略

统计模型调用次数。DND 通常应一次打分并按预测降序删除；R-ID、R-OD、R-SD 和 PageRank 必须每次删除后重算。

### 数据泄漏

对图的边集做哈希，并检查 seed/文件，发现训练测试重复。确保标签或测试指标没有进入输入特征，也没有超出论文描述被用于模型选择。

## 模型结构专项

必须从最终生效配置生成逐层对照表，至少包含：

| 字段 | 必查内容 |
|---|---|
| 输入 | 节点特征名称、数量、归一化、输入 shape |
| GAT | 层数、每层输入/输出通道、head 数、concat/mean、自环、边方向、dropout |
| 残差 | 残差来源、线性投影、shape 是否对齐、加法位置 |
| 激活/归一化 | 每层 ELU 或其他激活、BatchNorm/LayerNorm、顺序 |
| MLP | 各层实际宽度、激活、dropout、最终输出维度 |
| 输出 | Sigmoid 是否在模型内、每节点还是每图输出、排序使用的张量 |
| 损失 | 函数/公式、logits 或概率输入、soft label 兼容性、reduction/weight |
| 参数 | 总参数量、不同网络族/规模是否实例化不同模型 |

必要时注册 forward hook 或使用框架 summary；输出必须来自一次真实前向传播或明确标注为纯静态推断。重点识别：

- 多头拼接导致的维度乘以 head 数，却被误写成论文的通道数。
- GAT 库默认双向化、自环或消息方向与论文入邻居语义不一致。
- 残差投影未使用、广播相加或 silently 截断。
- 论文要求 ELU，但实现使用 ReLU/GELU 或漏掉层间激活。
- 模型内 Sigmoid 后又使用 `BCEWithLogitsLoss`，造成重复 Sigmoid；或输出 logits 却用要求概率的 loss。
- 将 `[0,1]` 软标签阈值化，改变最优集合频率含义。
- 配置声明与实际构造不一致，或 checkpoint 与当前模型不匹配。

## 实验结果专项

对每个论文主结果记录：论文结论、复现数值/曲线、统计口径、差异、可能原因和证据。至少检查：

- ER/SBM/SPL 各规模的 AUC 排名是否保持“DND 优于 degree/PageRank、弱于 CoreHD”的总体趋势。
- DND 计时是否仅含预测与拆解，并在规模增大时保持论文所述优势。
- 四阶聚合、多特征消融是否复现论文的通常优势，而非只展示有利案例。
- 真实网络是否覆盖论文数据集；未覆盖时不要泛化“复现成功”。
- 随机方法是否按论文次数重复，是否报告均值、方差/误差，是否固定 seed。
- 图片、CSV、日志、checkpoint 与执行命令/配置能否一一对应。

若只有截图、没有原始数值和生成链，结论最多为“外观一致、结果不可审计”。若结果不一致，先排查数据、标签、方向、指标和统计口径，再归因于随机性。

## 结果解释

- 分开评价“论文忠实度”和“实验可信度”。
- 曲线外观相似不能证明数据、标签或指标实现正确。
- 数值不同不必然是缺陷，因为论文没有提供随机种子及多个生成/训练细节。
- 成功的小型语义测试比匹配类名或函数名更有证明力。
- 若依赖或数据集阻塞执行，完成静态审查并列出准确解阻步骤。
- 优点也需要证据：例如定向测试通过、配置完整、结果可重建、主动分析论文歧义或设计有效额外消融。
